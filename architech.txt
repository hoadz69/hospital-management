Clinic SaaS / Hospital Management Architecture Notes

File này là index kiến trúc ngắn. Báo cáo đầy đủ nằm trong `clinic_saas_report.md`.

1. Sản phẩm

- Nền tảng Clinic SaaS multi-tenant.
- Owner Super Admin quản lý tenants, plans, domains, templates, publishing, reports.
- Clinic Admin quản lý một tenant/phòng khám.
- Public Website render content và booking flow theo tenant.

2. Frontend

- Vue 3 + Vite + TypeScript.
- Apps:
  - apps/public-web
  - apps/clinic-admin
  - apps/owner-admin
- Shared packages:
  - packages/ui
  - packages/design-tokens
  - packages/api-client
  - packages/shared-types
- UI source of truth là Figma/FigJam qua MCP khi tool có sẵn.

3. Backend

- API Gateway/BFF đứng trước services.
- Services:
  - identity-service
  - tenant-service
  - website-cms-service
  - template-service
  - domain-service
  - booking-service
  - catalog-service
  - customer-service
  - billing-service
  - report-service
  - notification-service
  - realtime-gateway
- Mỗi service theo Clean Architecture:
  - Api
  - Application
  - Domain
  - Infrastructure

4. Data

- PostgreSQL: relational và transactional data.
- MongoDB: CMS/page/template/layout JSON.
- Redis: cache, tenant config, domain mapping, rate limit, slot lock.
- Kafka/Event Bus: async domain events.
- SignalR/WebSocket: realtime gateway về sau.

5. Multi-Tenant Mandatory Rules

- Mọi tenant-owned command/query phải có tenant context.
- Clinic Admin chỉ truy cập tenant của mình.
- Owner Super Admin mới được cross-tenant.
- Public Website resolve tenant bằng domain/subdomain.

7. Nội Dung Đã Xác Nhận Từ PDF

- Source of Truth PDF xác nhận product map: Owner Super Admin, Clinic Tenant, Clinic Admin, Public Website, Create Clinic Flow, Domain Management, Tenant Management, Global Template Library, Template Config, Plan and Billing.
- Create Clinic Flow: input clinic info, choose package, create default subdomain, choose template, publish and send login.
- Domain Management: default subdomain, custom domain, DNS verification, SSL certificate, publish website.
- Clinic Admin modules: dashboard, appointments, homepage slider, website settings, staff/schedule, services/pricing, payments/reports, customers/records.
- Public Website modules: homepage, booking flow, contact/map, staff pages, service pages, mobile website.
- Technical PDF xác nhận flow: Users -> CDN/Edge/WAF -> Frontend Apps -> API Gateway/BFF -> services -> PostgreSQL/MongoDB/Redis/Kafka.
- Clean Architecture trong mỗi .NET service: API Layer, Application Layer, Domain Layer, Infrastructure Layer.
- Realtime Gateway là socket future cho live booking, notifications, dashboards.
